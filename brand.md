# هویت بصری کامل پلتفرم کتاب‌خوان دوزبانه هوشمند

نام پیشنهادی برای این هویت بصری:

# **MirrorRead / آینه‌خوان**

ایده مرکزی برند این است:

> **کتاب اصلی و ترجمه، مثل دو صفحه روبه‌روی هم؛ دقیق، آرام، هم‌زمان و قابل اعتماد.**

این هویت بصری باید حس‌های زیر را منتقل کند:

* حس کتاب، مطالعه و تمرکز.
* حس تکنولوژی هوشمند، اما نه سرد و ماشینی.
* حس ترجمه دقیق و قابل اتکا.
* حس دوگانگی هماهنگ: متن اصلی / متن ترجمه، چپ / راست، زبان مبدا / زبان مقصد.
* حس آرامش برای مطالعه طولانی‌مدت.
* حس کنترل برای کاربر حرفه‌ای: cache، queue، glossary، مدل AI، هزینه و تنظیمات.

---

# 1. شخصیت برند

## 1.1 جایگاه برند

**MirrorRead** یک ابزار ساده ترجمه نیست؛ یک محیط خواندن دوزبانه است.

جایگاه پیشنهادی:

> یک کتاب‌خوان هوشمند برای خواندن عمیق کتاب‌های زبان اصلی، با ترجمه فارسی ساختارمند، همگام و قابل کنترل.

## 1.2 شخصیت برند

| ویژگی     | توضیح                                                               |
| --------- | ------------------------------------------------------------------- |
| آرام      | فضای بصری نباید شلوغ، پرنور یا فریادزن باشد.                        |
| دقیق      | همه چیز باید حس نظم، alignment و اعتماد بدهد.                       |
| کتاب‌محور | UI نباید شبیه داشبورد خشک SaaS شود؛ باید هنوز حس مطالعه داشته باشد. |
| هوشمند    | AI باید حضور داشته باشد، اما نه به شکل اغراق‌شده یا فانتزی.         |
| حرفه‌ای   | مناسب دانشجو، مترجم، پژوهشگر، زبان‌آموز جدی و خواننده تخصصی.        |
| گرم       | چون مطالعه طولانی است، رنگ‌ها نباید کاملاً سرد و corporate باشند.   |

## 1.3 شعارهای پیشنهادی

### فارسی

* **متن اصلی و ترجمه، روبه‌روی هم.**
* **زبان اصلی را بخوان، فارسی بفهم.**
* **کتاب را ترجمه نکن؛ دوزبانه بخوان.**
* **خواندن عمیق، ترجمه همگام.**
* **هر صفحه، دو زبان، یک تجربه.**

### انگلیسی

* **Read across languages.**
* **Original and translation, side by side.**
* **A smarter way to read in two languages.**
* **Context-aware bilingual reading.**

شعار اصلی پیشنهادی:

> **متن اصلی و ترجمه، روبه‌روی هم.**

# 2. نام، لوگو و نماد برند

## 2.1 نام اصلی

### انگلیسی

**MirrorRead**

معنی: خواندن آینه‌ای، خواندن همراه با بازتاب ترجمه.

### فارسی

**آینه‌خوان**

این نام برای UI فارسی، مارکتینگ فارسی و نمایش در لوگوی فارسی مناسب است.

## 2.2 ساختار لوگو

لوگو باید از سه مفهوم ساخته شود:

1. **کتاب باز**
2. **آینه / دو پنل روبه‌رو**
3. **هم‌راستایی متن اصلی و ترجمه**

## 2.3 ایده نماد لوگو

نماد پیشنهادی:

```txt
دو صفحه روبه‌روی هم
با یک خط مرکزی شبیه spine کتاب
و دو خط متن در هر سمت
که یکی چپ‌چین و دیگری راست‌چین است
```

### فرم تصویری پیشنهادی

* یک مستطیل باز شبیه کتاب.
* سمت چپ: خطوط کوتاه افقی LTR.
* سمت راست: خطوط افقی RTL.
* وسط: یک خط عمودی نرم.
* فضای منفی وسط می‌تواند حس حرف **M** یا آینه را ایجاد کند.

## 2.4 نسخه‌های لوگو

### نسخه اصلی

```txt
[Icon] MirrorRead
       آینه‌خوان
```

### نسخه افقی

```txt
[Icon] MirrorRead
```

### نسخه فارسی

```txt
[Icon] آینه‌خوان
```

### نسخه کوچک

فقط آیکون کتاب/آینه.

## 2.5 ابعاد پیشنهادی لوگو

| کاربرد               | اندازه               |
| -------------------- | -------------------- |
| Favicon              | 32×32                |
| App icon             | 512×512              |
| Sidebar logo         | 32px icon + wordmark |
| Header logo          | 40px height          |
| Splash / Empty State | 96px تا 144px        |

## 2.6 فضای امن لوگو

فضای خالی اطراف لوگو باید حداقل برابر با عرض spine مرکزی آیکون × 4 باشد.

قانون ساده:

```txt
Clear Space = ارتفاع حرف M در MirrorRead
```

## 2.7 استفاده‌های ممنوع

نباید:

* لوگو کشیده یا فشرده شود.
* روی تصویر شلوغ بدون overlay قرار بگیرد.
* از رنگ‌های alert برای لوگو استفاده شود.
* لوگو با gradientهای فانتزی و پررنگ نمایش داده شود.
* در Reader، لوگو بزرگ و مزاحم نمایش داده شود.

| Paragraph     | ۳ تا ۵ خط                |
| Quote         | block با border          |
| Table         | grid skeleton            |
| Image caption | خط کوتاه زیر placeholder |
| Footnote      | متن کوچک‌تر              |

رنگ skeleton در ترجمه:

```txt
Base: #FCE7B8
Highlight: #FFF3D6
```

انیمیشن:

```css
animation-duration: 1.4s;
easing: ease-in-out;

margin-inline-end
padding-inline-start
padding-inline-end
border-inline-start
border-inline-end

text-align: end
```

```css
margin-left
margin-right
padding-left
padding-right
border-left
border-right
```

مگر جایی که واقعاً جهت فیزیکی مهم است.

## پنل‌ها

* Original English: `dir="ltr"`

* UI فارسی: `dir="rtl"`

# 20. Accessibility

## الزامات

* contrast متن reader باید بالا باشد.
* focus state همه کنترل‌ها واضح باشد.
* رنگ تنها نشانه وضعیت نباشد؛ badge text و icon هم لازم است.
* keyboard navigation برای reader actions.
* escape برای بستن popover/modal.
* کاهش motion برای کاربران حساس.
* target touch حداقل 44px.
* font size reader قابل افزایش باشد.
* line height قابل تغییر باشد.
* حالت dark و sepia داشته باشد.

## Focus Ring

```css
:focus-visible {
  outline: none;
  box-shadow: 0 0 0 3px #D7E1F2;
}
```

در dark:

```css
box-shadow: 0 0 0 3px rgba(143, 174, 232, 0.35);
```

# 32. Design QA Checklist

قبل از پیاده‌سازی هر صفحه، این چک شود:

```txt

آیا skeleton ساختار متن را حفظ می‌کند؟
آیا دکمه AI بیش از حد فانتزی نیست؟
```

# 33. جمع‌بندی هویت بصری

هویت بصری **MirrorRead / آینه‌خوان** باید بین سه جهان تعادل برقرار کند:

```txt
کتاب و مطالعه
+
ترجمه و زبان
+
هوش مصنوعی و کنترل فنی
```

فرمول بصری نهایی:

```txt
Paper warmth
+ Ink precision
+ Amber translation highlight
+ Calm AI purple
+ Split-pane reading structure
```

نتیجه باید محصولی باشد که وقتی کاربر آن را باز می‌کند، حس کند وارد یک ابزار شلوغ AI نشده؛ بلکه وارد یک **محیط مطالعه دقیق، آرام و هوشمند** شده است.

هویت پیشنهادی نهایی:

```txt
نام: MirrorRead / آینه‌خوان
شعار: متن اصلی و ترجمه، روبه‌روی هم
رنگ اصلی: Midnight Ink
رنگ مکمل: Warm Amber
حس بصری: کتابی، آرام، دقیق، هوشمند
UI: خلوت، ساختارمند، دوزبانه
Reader: Sepia-first، قابل تغییر به Light/Dark
AI: کاربردی، کنترل‌شده، غیرنمایشی
```

این Design System از همان ابتدا برای پیاده‌سازی واقعی در Next.js و Tailwind قابل تبدیل به token، component و Figma library است.
